## Usage

### Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.5.0
- [AWS CLI](https://aws.amazon.com/cli/) configured with appropriate credentials
- AWS account with sufficient permissions

### Quick Start

```bash
# 1. Configure AWS credentials
export AWS_ACCESS_KEY_ID=your-key
export AWS_SECRET_ACCESS_KEY=your-secret
export AWS_DEFAULT_REGION=us-east-1

# 2. Initialise Terraform
terraform init

# 3. Review the plan
terraform plan

# 4. Apply (creates resources in your AWS account)
terraform apply
```

Or use the included `Makefile`:

```bash
make init     # terraform init
make plan     # terraform plan
make apply    # terraform apply -auto-approve
make destroy  # terraform destroy
```

## Known limitations

- **Cross-module containment wiring is automated.** When a diagram shows
  one resource nested inside another (e.g. an EC2 instance inside a subnet)
  and both land in the *same* Terraform module, the reference is wired
  directly (e.g. `subnet_id = aws_subnet.x.id`). When they land in
  *different* modules (e.g. the instance in `compute/`, the subnet in
  `networking/`), the wiring is still real — the parent module gets an
  `output` for the referenced resource's `id`, the child module gets a
  matching `variable`, and the root `main.tf` passes it through the module
  call. A `NOTE:` comment above the resource block documents where each
  cross-module value comes from — search for `NOTE:` if you want to see
  all of them:

  ```bash
  grep -rn "NOTE:" .
  ```

- **Passwords & secrets:** resources needing a master password (RDS, Aurora,
  Redshift) use the provider's `manage_master_user_password` option instead
  of a literal password value, so no secret ever appears in generated code —
  AWS Secrets Manager creates and stores it automatically on `apply`.
- **AMI IDs:** EC2 AMI IDs are region-specific. Find the correct one for
  your region at: https://aws.amazon.com/amazon-linux-ami/
- **Backend state:** The `backend.tf` file contains a commented-out S3
  remote state config. Uncomment and configure it for team use.
